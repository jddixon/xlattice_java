Overview

JML is a utility for producing HTML from simple text with a minimum of markup. By default it actually generates XML in Anakia format. The XML output files are then merged with standard templates to generate Web site pages.

Command Line

JML will normally be run in the jml subdirectory of a project's base directory. By default input files have a .jml extension and JML will write its output to a file with the same base name and a .html extension in the xdocs subdirectory of the base directory.

As an example, if under UNIX/Linux you type

    cd jml
    jml foo

JML will read jml/foo.jml and write its output to xdocs/foo.xml.

There are a number of command line options. Typing

    jml -h

will give you a brief description of those currently available.

Markup

Title and Author

The utility expects the first line of the file to be its title. The author may optionally be specified in the second line by an expression line

    %author(fred, fred@freddy.com)

Whatever precedes the comma is interpreted as the author's name. Whatever follows it is an email address.

Section Headings

Section and subsection titles appear on lines by themselves with one or sharp signs (#) respectively as the first character(s). So

# Gone with the Wind

is translated into

<section name="Gone with the Wind">

Only two levels are currently supported.

Text Font Selection

This level of markup is done by bracketing text with single characters representing the kind of formatting wanted. For example, *make me bold* generates make me bold. The set currently available is

charresult
*starmake bold
_underscoreitalicize
`left single quoteuse teletype font, usually for code

These markups may not be nested. Expressions like *_whatever_* will have an unpredictable result.

Excaping Formatting Characters

Any characters with special meaning to JML can be used normally by preceding them with a backslash.

Lists

HTML's ordered and unordered lists are produced by writing 'o ' or '1 ' (lower case letter O and the digit 1) at the beginning of a line. So

o foo
o bar

will generate

<ul>
<li>foo</li>
<li>bar</li>
</ul>

which becomes

  • foo
  • bar

when rendered by the browser.

Lists may not be nested.

Blocks

If you want text to just be passed through unchanged, begin the block with {{ at the left margin and end it with }}. Whatever occurs between these becomes Anakia <source> blocks, appearing in the output in a teletype font and inside a box.

Blockquotes

Marked by beginning a line with space-doublequote ( ") and ending the blockquote with another doublequote (").
